Classificació d’objectes

  • Objectes definits per l’usuari: els defineix i els crea el programador.
  • Objectes natius de JS: (o predefinits)
    1. Objectes associats al tipus de dades primàries: String, Number, Boolean
    2. Objectes associats a tipus de dades compostes: Array, Objecte
    3. Objectes que proporcionen utilitats: Date, Math, RegExp
  • Objectes del navegador: BOM(Browser Object Model). Permeten manipular les finestres i interaccionar amb l’usuari, els més destacats:
    1. Window
    2. Navigator
    No són estàndards
  • Objectes del document: DOM(Document Object Model). Defineixen l’estructura de la interfície de la pàgina HTML. El W3C ha dut a terme una especificació estàndard del DOM

Date

Els objectes Date una vegada creats no canvien, el seu valor és estàtic. Nomes es pot modificar mitjançant els mètodes de la classe Date (setter’s)

Les dates en JS es calculen en mil•lisegons a partir de l’1 de Gener de 1970 a les 00:00 seguint el Coordinated Universal Time (UTC, en català Temps Universal Coordinat) i substituint al GMT l’any 1972(més info a https://ca.wikipedia.org/wiki/GMT)

String

Més info a: https://developer.mozilla.org/es/docs/Web/JavaScript/Reference/Global_Objects/String
Ajuda: https://flaviocopes.com/javascript-string/
La propietat cadena.length permet saber la longitud de la cadena de text.
En general, quan s'aplica un mètode sobre una cadena no es modifica la cadena original, sinó queretornen alguna cosa, una cadena transformada o bé un valor.
Per accedir a les posicions d’un String es pot fer de dues maneres:

        [], com si fos un array 
        amb el mètode .charAt(index)
El problema és que aquesta notació només dóna accés per llegir, no per modificar, doncs un String és immutable
Si s’intenta modificar un string no es produeix cap tipus de Warming o Error per part del navegador.

Això vol dir que el millor que es pot fer és crear una funció per poder modificar el contingut d’un string. Exemple d'alguns mètodes
  • str = r.replace(substr,noustr)
    s’utilitza per a reemplaçar la cadena substr per la cadena noustr.
    També pot substituir un sol caràcter, però només per la primera aparició.
    Es pot indicar una expressió regular(en el 1r paràmetre).
    Els dos paràmetres poden tenir diferent longitud.
    Si s’indica una expressió regular amb el modificador /g es pot reemplaçar totes les aparicions a la vegada.
    Exemple
  • array = str.split(separador,num)
    Permet convertir un String a un Array
    S’utilitza per a crear un array de cadenes a partir d’un string utilitzant un caràcter com a separador de la cadena original. El paràmetre opcional num és per indicar la quantitat de porcions que es volen.
  • str = substring(inici,fi)
    Retorna la subcadena que comença a la posició inici i acaba a la posició fi. El 2n paràmetre és opcional.
    Exemples:
    • cadena.substring(0,1) es queda amb l’element de la primera posició
    • cadena.substring(1,2) es queda amb l’element de la segona posició
    • cadena.substring(0,4) es queda amb els quatre primers elements (sense incloure el de la posició 4)

RegExp

Les expressions regulars permeten crear patrons a cercar, substituir, comptar, etc.
Més info a:https://developer.mozilla.org/es/docs/Web/JavaScript/Guide/Regular_Expressions
Exemple d'un parell de mètodes que es poden aplicar sobre Strings:

  • arrayNou = str.match(expressióRegular)
    retorna les coincidències dins la cadena str segons una expressió regular.
    Exemple d’expressió regular: /ain/gi
    Significat: cercar la cadena ain en str
    g = global, a tota la cadena, totes les coincidències
    i = indiferent si és majúscules o minúscules
  • posició = str.search(lletra/cadena/ExpressióRegular)
    cerca la 1a ocurrència d’una cadena o expressió regular i retorna la posició on l’ha trobat. És equivalent a str.indexOf(cad,0)

ALGUNES OBSERVACIONS SOBRE HTML

L'etiqueta input type="submit"

No s'ha de treballar a menys que s'utilitzi el servidor web. Si es treballa en local no s'ha de fer us del submit

Diferències sobre les etiquetes: <input type='button'> vs. <button></button>

Els botons creats amb l'etiqueta <button></button> són iguals que els botons creats amb l'etiqueta <input type='button'>, però ofereixen possibilitats de representació més rics. L'element BUTTON pot tenir contingut. Per exemple, una imatge, i fins i tot amb algun efecte, cosa que el INPUT no permet. A part, a nivell de sintaxi l’etiqueta button sempre ha d’anar acompanyada de la seva corresponent etiqueta de tancament, en canvi un input no té etiqueta de tancament.
