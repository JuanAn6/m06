Selecció de llenguatge de programació de clients web

JavaScript

Encara que alguns elements de sintaxis recordin a JAVA, no té res a veure, es va escollir aquest nom per raons de màrqueting, doncs estava de moda.

És un llenguatge basat en l’estàndard ECMAScript de ECMA (organització diferent de W3C)

Hi ha lleugeres diferències en la implementació de JS dels navegadors, encara que actualment tots són bastant compatibles entre sí.

Inicialment era un llenguatge interpretat, però actualment s’executa amb màquines virtuals en els navegadors (guanyant en velocitat d’execució i eficiència de memòria)

Característiques de Javascript

  • Interpretat: no requereix compilació, l’ intèrpret és el navegador de l’usuari.
  • Tipat dèbil (tipus de dades) / dinàmic: és una característica habitual en els llenguatge de script), no cal declarar el tipus de la variable abans de fer-la servir.
  • Orientació a objectes: basats en prototipus
  • Orientat a esdeveniments: fer clic, arrossegar un objecte, sortir d’una caixa de text,...

Creen i gestionen aplicacions de manera completa: Angular.js, React, Vue, Backbone.js, Ember,...

Declaració de dades: VAR, LET o CONST

  • Les constants: const. Quan no es vol deixar modificar el valor d'una dada.
  • Les variables globals: var
  • Les variables: let. Molt mé adequat que VAR, doncs no deixa crear de nou una variable amb el mateix nom.
Recomanacions de Google respecte var,let i const:

No usar var mai més, només const per les constants i let per les variables locals o globals.


		let num1;
Declarar les constants amb majúscules i amb guions baixos per separar paraules: 

		const NUMBER = 5;
		const MAX = 100;

Cada línia és per declarar una única variable:


		// malament
		let a = 1, b = 2, c = 3;
		// correcte
		let a = 1;
		let b = 2;
		let c,d,e;

Diferències entre VAR, LET i CONST. Resum final interessant. En anglès.

Noms de les variables

JS és case-sensitive ( nomPersona != nompersona != nom_persona) El nom d’una variable ha de complir les següents normes:

  • Només pot estar format per lletres, números i símbols $ i guió baix (_)
  • El primer caràcter no pot ser un número
  • No pot tenir espais ni cap signe de puntuació
  • No poden ser paraules clau/reservades pel propi llenguatge

L'etiqueta < noscript >

Alguns navegadors no disposen de suport complet de JavaScript, altres navegadors permeten bloquejar-lo perquè creuen que la navegació serà més segura.
L’etiqueta < noscript > s’utilitza per a detectar aquests tipus de navegador i així poder oferir un missatge a l’usuari indicant que ha d’activar JavaScript.

Tipus de dades

En JS qualsevol valor numèric diferent de 0 és TRUE. I el 0 és FALSE.
Per cadenes / strings, qualsevol que tingui contingut es considera TRUE, menys la cadena buida que és FALSE.
I qualsevol variable declarada però no definida (assignada a un valor) és FALSE.

Assignacions

Declaració d’una variable:


		let edat;
variable declarada però no definida, per tant, el seu tipus és undefined i el seu valor també. Si és necessari s'ha de comparar amb la paraula undefined sense cometes.

Assignació d’un valor a una variable, definició d’una variable

		edat = 23;
JS permet que el tipus d’una variable sigui dinàmic, és a dir, la variable edat pot canviar de tipus simplement assignant-la a un valor no numèric

		edat = 'vint-i-tres';
Diferència entre cometes simples i dobles

Per JS no hi ha diferència i es poden intercalar dins de les cadenes de text.
Per escapar una cometa simple o doble es pot fer servir el símbol \ a davant.
seqüències d’escapament: \n (nova línia),\t (tabulador),\' (cometa simple), \" (cometa doble), \\ (barra inclinada)
És adequat acostumar-se a delimitar les cadenes de text amb la cometa simple, abans que amb la doble.

Sentències

  • document.write no s'ha de fer servir mai
  • return: s’indica dins d’una funció per poder retornar un valor o simplement per sortir de la funció
  • break: força la interrupció de l’execució d’un bucle
  • continue: interromp l’execució de la iteració actual i transfereix el control a la iteració següent
Mètodes de l'objecte Window: alert, confirm, prompt

  • window.alert o alert genera una petita finestra d'avís.
  • window.confirm o confirm genera una petita finestra amb 2 botons. Si es prem OK retorna true, en cas contrari false.
  • window.prompt o prompt genera una petita finestra amb 2 botons i una caixa de text, es pot indicar un segon paràmetre pel contingut per defecte de la caixa de text. El valor recollit de la caixa de text per defecte sempre serà String, cal fer ús dels mètodes parseInt o parseFloat.

Es tracta de les estructures condicionals treballades en altres llenguatges:

  • Estructura condicional doble: if - else
  • Estructura condicional multiple: switch . El valor que s'indica en cada case pot ser: un valor enter, un caràcter o bé una cadena.
  • Operador ternari: exemples
Recordar que l'ús de les {} claus és opcional quan hi ha una única instrucció en un bloc.

Bucles

En JS existeixen diferents tipus de for. Per recorrer un array es pot fer servir el for-of: 


		let llista = [3,5,7,9];
		for (elem of llista)
			{
				console.log(elem); 
			}
