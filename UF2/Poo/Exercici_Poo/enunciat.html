Classe Persona

Crear una nova classe Persona amb les propietats:

  • nom
  • cognom1
  • cognom2
  • edat
  • sexe
  • estat civil

La classe Persona ha de tenir els mètodes:

  • validarEstat(): Permetrà comprovar que l’estat està entre un dels següents valors: Solter, Casat, Separat, Divorciat o Vidu. I si és així, assignarà el valor a l'objecte en majúscules.
  • toString(): retornar una representació textual d'una Persona

Al premer el botó cal mostrar la invocació del mètode toString de la persona creada a l'etiqueta amb id='dadesPer'.

Fer un mínim de validació de les dades entrades del formulari per poder crear una Persona.
Les dades obligatòries són: nom, cognom1, edat, sexe i estat civil.
Si no s'indica cognom2 es posa a null.
L'edat ha de ser positiva.
En el constructor invocar el mètode validarEstat.

Classe Cotxe

Crear una nova classe Cotxe amb les propietats:

  • matricula
  • color
  • marca
  • model
  • data matriculació

Aquesta classe ha de tenir un mètode anomenat veureParcial que retorni una cadena amb el color,la marca i el model de l'objecte que l'invoca.

Programar el mètode toString que aprofiti el mètode veureParcial

En prémer el botó per crear un objecte Cotxe i visualitzar-lo dins l'etiqueta dadesCotxe.

El camp per indicar la data de matriculació és un "input='date'" el qual retornarà un String que es pot passar com a paràmetre en el constructor d'un objecte "Date" per crear la data, si la data no és correcte generarà "Invalid Date".

Classe LLIBRE

Crear una nova classe Llibre amb les propietats:

  • títol. Sempre s'ha de desar en majúscules
  • autor. Si es passa null, undefined o cadena buida es posarà el valor "Desconegut"
  • pàgines totals
  • pagines llegides (inicialitzada a 0, només es modifica via un mètode indicat posteriorment)

La classe Llibre ha de tenir els mètodes:

  • llegir(): Si no s’indica paràmetre cal fer que l’objecte ja s’ha llegit tot, en cas contrari cal validar que el paràmetre és numèric, positiu i menor o igual que les pàgines totals del llibre. Si el valor no és vàlid, el mètode no farà res. En cas que ja s'hagin llegit pàgines i el valor indicat + les pàgines llegides superi el total, s'assignarà directament el total com a pàgines llegides.
  • toString()

Es pot suposar (per simplificar) que un llibre s'identifica pel títol.

Cada cop que es crei un nou llibre cal afegir el toString dins el "select".

Cada cop que un llibre ja s'hagi llegit del tot ha de desaparèixer del select i es visualitzarà després dels formularis en la zona amb id sol3

Per fer el què es demana, es recomana crear un array d'objectes llibre, i quan ja estiguin llegits, esborrar-los de l'array.
